# Security Policy for YATO

We take the security of the YATO platform seriously. This document outlines our policy on security, supported versions, and the process for reporting potential vulnerabilities.

## Supported Versions

Only the latest major and minor releases receive active security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in YATO, please do **not** open a public issue. Instead, report it privately through one of the following methods:

- **Email**: Send a detailed description of the vulnerability to [support@honet.web.id](mailto:support@honet.web.id).
- **GPG Key**: If you wish to encrypt your email, you can request our security team's GPG public key.

Please include the following information in your report:

1. **System & Components**: The versions of YATO and related containers (frontend, backend, database) where the vulnerability is present.
2. **Impact**: A description of the potential impact of the vulnerability.
3. **Proof of Concept (PoC)**: Detailed steps, code snippets, or configurations required to reproduce the vulnerability.

## Our Security Response Process

Once a vulnerability report is received, the YATO security team will:

1. **Acknowledge**: Respond to your report within **48 hours** to acknowledge receipt.
2. **Investigate**: Conduct a thorough investigation to confirm the vulnerability and assess its impact.
3. **Patch**: Develop a fix/patch to resolve the vulnerability.
4. **Coordinate**: Coordinate a release date with you and publish a security advisory (if applicable) once a secure update is ready.

Thank you for helping us keep YATO secure for everyone!
